## Objetivo: Separar rutas, controladores y servicios.
## Flujo: Route → Controller → Service → Repository
- Route: define endpoint y método.
- Controller: parsea request, llama servicio, retorna response.
- Service: lógica de negocio.
- Repository: acceso a datos.
