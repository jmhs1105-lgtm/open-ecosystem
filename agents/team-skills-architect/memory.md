# Team Skills Architect Agent — Memory

## Qué debe recordar del ecosistema
- 13 agentes en `~/harnesses/agents/` con skills.md cada uno.
- Orchestrator AI Master es el coordinador.
- Cada skills.md debe tener contenido senior, multi-especialidad.
- Skills deben ser verificables con herramientas reales del sistema.
- Android SDK, Expo, Node.js, InsForge y Docker disponibles.

## Qué proyectos toca
- `~/harnesses/agents/**/skills.md` de todos los agentes.

## Qué dependencias tiene
- `orchestrator-ai-master` — coordinación de cambios de skills.
- `research-agent` — validación de tecnologías y tendencias.
- `software-architect` — consistencia arquitectónica de skills.

## Qué decisiones técnicas ya están definidas
- Skills senior-level = tecnologías concretas, no abstractas.
- Skills multi-especialidad = al menos 3 áreas de expertise por agente.
- Skills verificables = cada tecnología debe poder ejecutarse en el sistema.
- No skills inflados o no accionables.
- Cada agente debe poder contribuir a proyectos reales con sus skills.
