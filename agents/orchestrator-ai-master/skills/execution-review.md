# Skill: execution-review
## Agente dueño: Orchestrator AI Master
## Categoría: Core / Quality
## Objetivo: Evaluar si una tarea quedó realmente completa o solo documentada.
## Cuándo usarla: Después de que un agente reporta "completado".
## Pasos seguros
1. Revisar si el código compila/ejecuta.
2. Revisar si hay tests.
3. Revisar si la documentación está actualizada.
4. Revisar si hay errores residuales.
5. Si solo documentado sin implementar: reportar.
## Señales de tarea incompleta
- Solo se creó README sin código.
- Solo se documentó sin probar.
- Tests no ejecutados.
- Validación no realizada.
