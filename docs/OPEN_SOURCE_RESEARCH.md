# Open Source Research

## Research Process

1. Define question
2. Search official docs (not AI guesses)
3. Compare alternatives (pros/cons)
4. Cite sources
5. Recommend with reasoning

## Tools

| Tool | Purpose |
|---|---|
| GitHub | Code search, issues, stars |
| npm | Package registry, downloads |
| Official docs | Reliable source |
| Stack Overflow | Community solutions |
| WebFetch | Read docs programmatically |

## Rules

1. No hallucination — cite source for every claim
2. Official docs over blog posts
3. Check last commit date (< 6 months)
4. Check license (prefer MIT, Apache 2.0)
5. Check bundle size for frontend packages
6. No paid software unless justified
