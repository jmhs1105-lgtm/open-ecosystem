# Skill: AppFlowy Workspace

## Objective

Organize the OpenCode ecosystem using AppFlowy: projects, agents, skills, roadmaps, errors, and business ideas in a private open-source knowledge base.

## When to Use

- Documenting a new project
- Tracking agent status
- Recording errors and solutions
- Planning roadmap items
- Storing master prompts
- Brainstorming business ideas

## When NOT to Use

- Replacing git commits
- Replacing InsForge agent_state
- Replacing code documentation (.md files in repo)
- Storing secrets or API keys

## Agent Responsible

**Documentation & Knowledge Team**

## Related Tools

- AppFlowy (appflowy.io)
- Markdown docs (in repo)

## Workspace Structure

See `docs/APPFLOWY_WORKSPACE_PLAN.md`

## Rules

1. No secrets or API keys in AppFlowy
2. Keep in sync with repo docs
3. Use templates for consistency
4. Archive completed items, don't delete
