# Skill: UI/UX Pro Max

## Objective

Apply professional design methodology to every screen: research, user flows, wireframes, prototypes, visual design, component systems, states, accessibility, and conversion optimization.

## When to Use

- Creating any new screen
- Redesigning existing screens
- Setting up design system
- Reviewing UI quality
- Starting a new project

## When NOT to Use

- Backend development
- Bug fixes
- Documentation
- CLI tools

## Agent Responsible

**Team Skills Architect**

## Related Teams

- Design & UI Team
- Mobile & Android Team
- React Native Expo Team

## Related Tools

- Penpot (design)
- Expo / React Native (implementation)
- VS Code (code)
- AppFlowy (documentation)

## Phases

1. **Research**: User, problem, goal
2. **User Flow**: Entry → Steps → Success/Error
3. **Wireframe**: Layout structure, no colors
4. **Prototype**: Interactive mockup
5. **Visual Design**: Colors, typography, spacing
6. **Components**: Reusable, documented, tested
7. **States**: Loading, empty, error, success, edge
8. **Accessibility**: Contrast, labels, touch targets
9. **Conversion**: Clear CTA, reduced friction
10. **Handoff**: Tokens, specs, flows to developer

## 10 Golden Rules

1. No programar pantalla importante sin user flow
2. No crear UI sin objetivo
3. No usar colores al azar
4. No copiar diseños protegidos
5. No priorizar belleza sobre claridad
6. Cada pantalla debe tener una acción principal clara
7. Siempre diseñar estados (loading, empty, error)
8. Touch targets ≥ 44pt
9. Color contrast ≥ 4.5:1
10. No más de 3 acciones por pantalla

## Validation

- [ ] User flow documented before coding
- [ ] All 10 phases completed
- [ ] Design tokens in code match Penpot
- [ ] All states designed
- [ ] Tested on real device
- [ ] Accessibility basics met
