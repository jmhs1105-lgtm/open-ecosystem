# Skill: VS Code Workflow

## Objective

Use VS Code as the visual editing companion to OpenCode AI for code review, debugging, and terminal operations.

## When to Use

- Reviewing code generated by OpenCode
- Running lint, tests, and builds in terminal
- Debugging with breakpoints
- Viewing git diffs before committing
- Editing files manually when needed
- Managing extensions and settings

## When NOT to Use

- As a replacement for OpenCode (use OpenCode for creation)
- For committing code (let OpenCode handle it)
- For running deploy commands (unless approved)

## Agent Responsible

**Dev Environment Team** (MCP Operations Agent)

## Related Tools

- VS Code
- Terminal (integrated)
- Extensions (see `.vscode/extensions.json`)

## Workflow

```
1. OpenCode creates/modifies files
2. VS Code opens workspace (~/harnesses/)
3. Review changes in Source Control panel
4. Check Error Lens for inline issues
5. Run npm run lint / npm test in terminal
6. Fix any issues manually or via OpenCode
7. OpenCode commits
```

## Configuration

- `.vscode/settings.json` — Editor preferences
- `.vscode/extensions.json` — Recommendations
